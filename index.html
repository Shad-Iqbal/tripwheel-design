v0.5.3-states Jun 27, 2026

Multi-photo stops, editable transit, share-to-promote-live

Polish + sharing push. Stops now hold multiple photos in a Facebook-style collage with a full-screen swipeable lightbox; the edit modal manages them with the same media uploader as the Social tool. Travel-time lines are editable, the builder header is simplified (back-over-title), the Share dialog is rebuilt to push the live link, and the dashboard Get Started guide gains a 4th step.

Multi-image Lightbox Editable transit Share live Onboarding
Update log
  • Get Started guide: added 4th step "Create an itinerary" → opens the builder (now "0 of 4 complete")
  • Stops support multiple photos — right-side collage (1/2/3/4+ with "+N"); click opens a full-screen lightbox with arrow keys, prev/next and touch-swipe
  • Edit modal uses the Social tool's media uploader (.thumb-strip): add / edit / remove photo tiles; images optional with clean placeholders
  • Transit lines (e.g. "17 min by metro") are now editable — click to set mode + minutes, or reset to the auto estimate; route totals respect overrides
  • Map pins remain draggable; transit recomputes against any manual overrides
  • Simpler header: back link + previous page sits above the title; removed the redundant "All trips" button
  • Share dialog rebuilt: two-column, live-itinerary preview on the left; right side promotes the live link (copy, Email, WhatsApp) with a secondary icon-only row to download / send a PDF copy

v0.5.2-states Jun 27, 2026

Editable days, editable stops & pins, responsive viewer

Control pass on the builder plus a viewer rebuild. Days can be added, retitled, re-dated and removed. Stop images are now optional and editable (clean category placeholder when none). Map pins are draggable to fix a location, with transit recomputing live. The shared viewer was rebuilt to be fully responsive across devices — space-optimized, cleaner, with minimal imagery.

Editable days Editable images Draggable pins Responsive viewer Polish
Update log
  • Days: Add day chip in the day tabs; per-day edit modal (title + date) with Remove day; auto-renumbered Day labels; empty-day state
  • Stop images now optional & editable — Image URL field in the edit modal + Remove image; cards fall back to a clean category-coloured placeholder instead of a forced photo
  • Map pins are draggable — drop a pin to set a stop's location; route line and transit estimates update live without resetting the view
  • Viewer fully rebuilt & responsive: fluid width (mobile → desktop), compact rows with small/optional thumbnails, tighter rhythm, day sections, sticky Reply CTA
  • New days seed a generic suggestion pool so "Draft this day" works on freshly added days

v0.5.1-states Jun 27, 2026

Trip Brief — capture the client's ask, power the AI

Fills the top advisor JTBD gap from the v0.5.0 critique: there was no way to record what the client actually wants, so the AI drafted generically. The New-itinerary flow is now unified around a short brief (travellers, budget, pace, interests, must-dos, constraints) with a "Start from: Blank / Template / AI" selector, and the brief lives on as an editable summary bar in the builder. The AI assist now reads the brief.

Trip Brief Intake JTBD AI from brief Advisor workflow
Update log
  • New-itinerary modal reworked: one brief form (name, destination, dates, client, travellers steppers, budget, pace, interests chips, must-dos, constraints) + a Start from selector (Blank / Template / AI)
  • Removed the old field-duplicating Blank tab; the brief is shared across all three start modes
  • Builder: persistent brief summary bar under the day tabs (party · budget · pace · interests) with Edit-brief, plus a calm nudge when empty
  • AI now reads the brief — "Draft this day" varies stop count by pace and orders suggestions by interests; "Generate draft" headlines the brief ("Tailored to 2 adults + 2 teens · relaxed · food")
  • Viewer cover shows a subtle "Planned for…" personalization line
  • Schema: additive itinerary.brief object — leaves hooks for the next jobs (lodging/transport, cost/pricing) without rework

v0.5.0-states Jun 27, 2026

Itinerary Builder tool — MVP added to the app shell

New Itinerary Builder feature lands as a real tool in the sidebar (no longer "Soon"). Ships the both-sides-valuable MVP: a client-trip library, an image-rich itinerary-first builder with a secondary collapsible map, a light "Draft this day" AI assist, and a polished, mobile-first no-login viewer with a Reply call-to-action. Built on v0.4.3's shell; reuses its tokens, toast, and modal system. Only new dependency is Leaflet.

Itinerary Builder MVP Library + Templates Map + images Shareable viewer Light AI
Update log
  • Sidebar: enabled "Itinerary Builder" across all screens with a New badge → switchScreen('itinerary')
  • Library: client-trip cards (photo covers + status pills), search, status chips (All / Drafts / Shared), plus a Templates tab and a grid ⇄ list view toggle
  • New-itinerary modal: three modes — Blank, Template (prefills from a chosen template), and AI draft
  • Builder redesigned image-first for scanability: large photo cards with category pills + durations, itinerary as the hero, map demoted to a secondary collapsible panel ("Map" toggle)
  • Stops are clickable (edit name/category/duration/note) and reorderable; manual "Add a stop" placed before the AI assist
  • Light AI: per-day "Draft this day" → 3 add/skip suggestions with images
  • Viewer: polished mobile-first shared page (cover, advisor card, timeline of image cards, transit) with a sticky Reply CTA + reply sheet
  • All feature JS namespaced under window.iti; reuses base toast() and .modal-overlay; scoped CSS under .iti-

v0.4.3-states Jun 24, 2026

Minimal dashboard — focused empty & active states

Dashboard stripped to what matters. Empty state is just the Get started card to cut cognitive load for new accounts; active state drops the coming-soon grid and redundant quick actions. Builds on v0.4.2's page-based topbar breadcrumbs and panel polish.

Minimal Empty states Dashboard Cognitive load Breadcrumbs
Update log
  • Dashboard empty state: reduced to greeting + single Get started card (560px). Removed zero-value stat row, empty Connected accounts panel, and "More tools" grid
  • Dashboard active state: removed 7-card "More tools on the way" grid and the redundant Quick actions panel (duplicated topbar Create post + Connected accounts)
  • Kept in active view: KPI row, Upcoming posts, Recent photo jobs, Connected accounts
  • Topbar: page-based breadcrumb hierarchy — path in .bc, current page in .tb-title only (no duplicate titles)
  • Fixed overlapping borders on nested empty states — strip frame from .panel > .empty-state
