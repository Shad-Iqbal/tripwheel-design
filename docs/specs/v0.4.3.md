# Tripwheel — Web UI Spec (v0.4.3)

Owner: Shad · shad@tripwheel.io  
Version: **v0.4.3-states** · Jun 24, 2026  
Companion prototype: `prototypes/v0.4.3-states.html`  
Supersedes: `docs/specs/v0.4.2.md` (delta only — unchanged screens inherit v0.4.2)

Scope of this revision: **Dashboard minimalism**. Empty and active states stripped to
what matters, to cut cognitive load. All other screens (Signup, Social Media, Photo Tool,
coming-soon tools, app shell, state machine) are unchanged from v0.4.2 — see that spec.

---

## Changelog (v0.4.2 → v0.4.3)

| Area | Change |
|---|---|
| Dashboard (new) | Reduced to greeting + single **Get started** card (max 560px). Removed the zero-value KPI row, the empty Connected accounts panel, and the coming-soon teaser grid |
| Dashboard (active) | Removed the 7-card **coming-soon grid** and the **Quick actions** panel (duplicated topbar Create post + Connected accounts) |
| Greeting copy (new) | "One step gets you going — connect a channel and you're set." (was "Let's get your workspace set up…") |
| Panel polish | Nested empty states inside a panel strip their own frame (`.panel > .empty-state`) — carried from v0.4.3 fork |
| Topbar | Page-based breadcrumb hierarchy retained from the fork; current page in `.tb-title` only |

Rationale: a new account does not need four "0" cards, an empty accounts panel, and a
roadmap grid competing with the one action that matters. The active dashboard kept only
recurring-value surfaces; the coming-soon grid and quick-action shortcuts were noise
(both reachable from the topbar / sidebar / connected-accounts panel).

## 2. Dashboard (revised)

Two layouts toggled by account state (Proto FAB: New vs Active).

### 2a. New account (onboarding) — minimal

```
DashboardPage · New
├── Topbar — Title "Dashboard" · CTA "Create post"
├── GreetingRow
│   └── "Welcome to Tripwheel, {Name} 👋" · "One step gets you going — connect a channel and you're set."
└── GetStarted (single column, max 560px)
    └── Panel · Get started
        ├── Header · "Get started" · "0 of 3 complete"
        └── ChecklistRow ×3 (numbered thumb · title · subtitle · action button)
            ├── 1 · Connect a social account → Connect (primary)
            ├── 2 · Schedule your first post → Create post
            └── 3 · Run your first photo job → Open Photo Tool
```

Removed vs v0.4.2: StatCardRow (4 zero cards), Connected accounts empty panel,
ComingSoonSection. The Get started checklist is the sole focal element.

### 2b. Active account — minimal

```
DashboardPage · Active
├── Topbar — Title "Dashboard" · CTA "Create post"
├── GreetingRow
│   ├── "Good morning, {Name} 👋" + date
│   └── WorkspaceSwitcher (avatar initial + company name)
├── StatCardRow (4 KPI cards — value + inline trend badge)
└── TwoColumnGrid
    ├── Col A (2/3)
    │   ├── Panel "Upcoming posts" → PostRow ×N
    │   └── Panel "Recent photo jobs" → JobRow ×N
    └── Col B (1/3)
        └── Panel "Connected accounts" → AccountRow ×N · "+ Connect another account"
```

Removed vs v0.4.2: ComingSoonSection (7-tool grid), Quick actions panel.
StatCard component unchanged — see v0.4.2 §2b.
